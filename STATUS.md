> **T-33 상세**: 원인은 전역 `pointerdown` 리스너가 `.btn`/`.skill` 클래스만 예외 처리하고 SHARE/COPY 버튼의 실제 클래스(`.share-btn`)는 누락 — 버튼 탭 시 `click` 핸들러(공유 로직)보다 `pointerdown`이 먼저 발동해 게임오버 화면의 재시작 로직(`primaryAction`)이 실행됨. `.share-btn`을 예외 목록에 추가해 해결. `shareScore`/`copyToClipboard` 자체 로직은 기존부터 정상이었음.

> **T-34 상세**: (1) COPY LINK가 SHARE SCORE와 동일한 전체 캡션 텍스트를 복사하던 버그 수정 — `SHARE_URL` 상수를 신설해 COPY LINK는 순수 링크(`https://playrevolv.netlify.app`)만 복사하도록 분리. (2) SHARE SCORE는 결과 이미지(REVOLV 로고+점수+CTA 카드)를 Web Share API 파일 공유로 우선 전달하도록 유지하되, 파일 공유 미지원 환경(주로 PC 브라우저)에서 텍스트만 공유되어 이미지가 누락되던 폴백 경로를 수정 — 이제는 이미지가 항상 새 탭에 열려 저장/공유 가능하고 캡션은 클립보드에 별도 복사됨.

> **참고(7/11)**: GA4 탐색에서 "사용자 의사 ID"를 측정기준으로 검색해도 결과 없음 — REVOLV 총사용자(47명)가 적어 GA4 개인정보 임계값(data thresholding)이 사용자 식별 차원을 숨기는 것으로 추정됨. 향후 유저 단위 세부 분석이 필요할 때 이 한계를 먼저 고려할 것 (대안: BigQuery Export 연동).

> **T-29 구현 상세(7/11)**: 시작화면·게임오버화면에 🏆 RANK 버튼 추가 → 상위100+내순위 리더보드 오버레이. 최초 게임오버 시(닉네임 없으면) 0.9초 후 닉네임 등록 오버레이 자동 노출, "나중에 하기" 선택 시 이후 자동 재노출 안 함(RANK 버튼으로 언제든 등록 가능). 닉네임 있으면 매 게임오버마다 조용히 자동 제출(신기록일 때만 서버가 갱신). 리더보드/닉네임 오버레이가 열려있는 동안 배경 탭으로 인한 재시작 오작동 방지 처리 포함.
